## 二、共通根因

多數「點了沒反應」的選單問題，根因是同一個：**中文字數比英文少，卻沿用了英文原本的空格排版，導致文字整片左移、離開遊戲固定的點擊判定框（hitarea）**——字看得到，但不在判定框上。正解是把中文對齊到英文原本的欄位位置。詳見 [除錯紀錄](BUGFIX_NOTES.md)。

## 三、平台包狀態

公開 repo 維持 **patch-only**（只放 `.patch`＋字型＋譯表，玩家跑 patched ScummVM，自備遊戲原檔）。含遊戲的完整包只在本機保留、不入 repo、不公開。

三平台 patch-only 均已更新到 GitHub Release（含選單對齊、片頭 Enter、讀檔殘留、防拷、佈告欄視覺修正）。

| 平台 | patch-only（GitHub Release）| 完整包（本機保留） |
|---|---|---|
| Linux (x86_64) | ✅ 已更新（含板圖清除）| ✅ 已重建・已開機驗證 |
| Windows (x64) | ✅ 已更新（含板圖清除）| ✅ 已重建 |
| macOS (universal arm64+x86_64) | ✅ 已更新（GitHub Action 重編）| ✅ 已重建（CI universal binary 本機注入遊戲檔）|

- macOS 只能在 macOS runner 上編（見 workflow 註解的踩雷防呆），故走 GitHub Action 產 universal binary，本機再注入遊戲檔組完整包。
- 對齊類修正（選單／能力值）只動譯表、重烘 `.tab` 即生效，不需重編引擎；片頭 Enter 與讀檔殘留是引擎修正，需重編。

## 四、如何取得修正

- **自行重建**：跑 `scripts/dev_setup.sh`，會抓乾淨 ScummVM v2.9.1、套上最新 `patches/agos-cht.patch`、編譯並放好字型譯表。詳見 [開發環境重建指南](DEV_SETUP.md)。
- **等釋出**：patch-only 平台包會更新到 GitHub Release。
