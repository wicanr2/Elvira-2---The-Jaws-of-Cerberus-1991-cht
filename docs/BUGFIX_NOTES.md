# 中文化除錯紀錄：疊層對齊與片頭跳過

> 收錄玩家實機回報的一批問題與修正。核心教訓只有一句：
> **中文比英文短，卻沿用英文的空格數排版時，文字會整片左移、離開遊戲固定的點擊判定框（hitarea），於是「看得到字卻點不到」。**
> 正解不是保留空格數，而是**把中文對齊到英文原本的欄位位置**。

## 一、根因：疊層文字與 hitarea 判定框錯位

Elvira 2 的選單（是／否、存檔／讀檔、暫停／結束）不是一般 UI 元件，而是**遊戲腳本定義的固定座標點擊框（hitarea）＋一行文字**。文字只是「畫在框上面」，真正決定「點了有沒有反應」的是那個看不見的框。

英文版的文字用空格排版，剛好落在各自的框中央：

```
    Yes             No          ← 英文：Yes 在左框、No 在右框
```

繁中版一開始沿用了同樣的空格數。但「Yes→是」「No→否」各短了兩個字元，整串往左縮，於是：

```
是               否            ← 中文沿用空格：字往左移,「否」離開了 No 框
└ 點這裡有反應      └ No 框在這裡(點框中間點到的是空白)
```

結果就是玩家回報的「點『否』沒反應」「點『結束／讀檔』沒反應」——字看得到，但它不在判定框上。

### 修正做法

把中文對齊到**英文字母原本的欄位（START 欄位）**，而不是保留空格數。以「是／否」為例：英文 `No` 的 `N` 落在第 19 欄，中文「否」就要補空格推到第 19 欄，才會落回 No 框：

```
       是           否          ← 「否」對齊到英文 No 的欄位 → 命中 No 框
```

驗證方式：在引擎 `boxController` 暫時加 log 印出點擊座標與命中的框 id，實測「點視覺上的『否』→ 命中 No box（id 0xC9）」才算對。修好後移除 debug。

> 座標細節：hi-res 疊層下滑鼠是 640×400，遊戲 hitarea 是 320×200，引擎會把座標 `>>1` 還原。所以視覺點 (405,355) → 折半 (202,177) → 命中 No 框（logical x186 y177 w18 h9）。

## 三、修改位置（都在 `patches/agos-cht.patch`，`// 非上游` gate）

- **片頭 Enter**：`engines/agos/metaengine.cpp` — Exit cutscene 動作補 RETURN／KP_ENTER 預設鍵位。
- **讀檔殘留**：`engines/agos/saveload.cpp` `AGOSEngine_Elvira2::loadGame` 結尾清 `_scaleBuf`。
- **選單／能力值對齊**：純譯表（`translations/zh.tsv` → 重烘 `elvira2_zh.tab`），**不必改引擎**。

> 對齊類修正只動譯表、重烘 `.tab` 即可生效，不需要重編引擎——這是 AGOS 繁中版「runtime 依 id 查表回填」設計的好處。

## 四、待實機確認（本文件不含修正）

以下三項屬遊戲物品／場景邏輯，headless 自動化難以導航重現，需玩家實機協助定位：

- **電梯鍵盤按鈕後按 R 卡住**：與防拷 bypass 應無關（玩家看得到並能操作鍵盤，代表該鍵盤不是被自動跳過的 SUB_1867／1869）。待確認英文原版是否也有同樣情形。
- **佈告欄清涼月曆拿不到**（YT 顯示應在手提箱內）、**拿下佈告欄物品後佈告欄仍顯示原物品**：屬遊戲物品狀態邏輯，與中文字串無關。
